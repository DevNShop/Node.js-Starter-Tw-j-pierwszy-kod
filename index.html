NODE.JS STARTER: TWÓJ PIERWSZY KOD

Czyli jak stworzyć własnego bota od zera, nie mając o tym zielonego pojęcia.

WSTĘP: CZYM WŁAŚCIWIE JEST BOT?

Wyobraź sobie, że bot na Discordzie to taki bardzo posłuszny, wirtualny pracownik. Problem w tym, że ten pracownik jest kompletnie ślepy, głuchy i nie zna żadnego języka. Nie potrafi niczego zrobić sam z siebie. Dopiero gdy napiszesz mu odpowiedni kod, dajesz mu "oczy" (żeby widział wiadomości), "uszy" (żeby słyszał komendy) i "usta" (żeby mógł odpisać).

Programowanie bota to po prostu pisanie instrukcji krok po kroku. W tym e-booku nauczysz się, jak złożyć swojego pierwszego wirtualnego pracownika. Gotowy? Zaczynamy!

ROZDZIAŁ 1: WARSZTAT I SILNIK (CO MUSISZ POBRAĆ)

Żeby cokolwiek zbudować, potrzebujesz narzędzi. W przypadku programowania botów potrzebujesz tylko dwóch darmowych programów.

1. Node.js (Silnik)

Node.js to taki "silnik", który sprawia, że nasz kod w ogóle zadziała. Bez niego twój komputer widziałby pliki z kodem jako zwykły, bezużyteczny tekst.

  • Wejdź na stronę: nodejs.org.
  • Pobierz i zainstaluj wersję z napisem "LTS". Instaluje się ją jak każdy program – po prostu klikaj "Dalej" (Next).

2. Visual Studio Code (Warsztat)

To jest twój stół roboczy. To specjalny notatnik stworzony dla programistów. Pokoloruje on twój tekst tak, abyś nie zgubił się w linijkach kodu.

  • Wejdź na stronę: code.visualstudio.com.
  • Pobierz i zainstaluj program.

ROZDZIAŁ 2: REJESTRACJA BOTA (DOWÓD OSOBISTY)

Zanim zaczniemy pisać, musimy powiedzieć Discordowi: "Hej, tworzę nowego bota!".

  1. Wejdź na stronę Discord Developer Portal i zaloguj się.
  2. Kliknij "New Application", nazwij bota i przejdź do zakładki "Bot".
  3. TOKEN BOTA: Kliknij "Reset Token". To długi ciąg znaków. To hasło do bota. Nigdy go nikomu nie dawaj!
  4. INTENTY: Na tej samej stronie (zakładka Bot) zjedź do "Privileged Gateway Intents" i włącz wszystkie trzy przełączniki (Presence, Server Members, Message Content). Kliknij "Save Changes".

ROZDZIAŁ 3: BUDUJEMY FUNDAMENTY

  1. Stwórz na pulpicie folder "moj-bot".
  2. Otwórz Visual Studio Code, kliknij File -> Open Folder i wybierz ten folder.
  3. Otwórz Terminal (Terminal -> New Terminal) i wpisz te dwie komendy:
    • npm init -y (tworzy teczkę na dokumenty bota).
    • npm install discord.js (kupuje botowi słownik języka Discorda).

ROZDZIAŁ 4: PISZEMY PIERWSZY KOD!

Stwórz nowy plik o nazwie index.js i wklej tam ten podstawowy szkielet bota:

// 1. IMPORTOWANIE NARZĘDZI I KONFIGURACJA
const { Client, GatewayIntentBits } = require('discord.js');
const TOKEN = 'TUTAJ_WKLEJ_SWOJ_TAJNY_TOKEN';

// 2. TWORZENIE BOTA
const client = new Client({
    intents: [
        GatewayIntentBits.Guilds,
        GatewayIntentBits.GuildMessages,
        GatewayIntentBits.MessageContent
    ]
});

// 3. KIEDY BOT SIE URUCHOMI...
client.on('ready', () => {
    console.log(`Zrobione! Bot zalogowal sie jako: ${client.user.tag}`);
});

// 4. KIEDY KTOS NAPISZE WIADOMOSC...
client.on('messageCreate', (message) => {
    if (message.author.bot) return;
    if (message.content === '!ping') {
        message.reply('Pong! Moj pierwszy kod dziala!');
    }
});

// 5. LOGOWANIE BOTA
client.login(TOKEN);

ROZDZIAŁ 5: ODPALAMY MASZYNĘ!

W terminalu (na dole VS Code) wpisz: node index.js

Jeśli widzisz napis "Zrobione!", wejdź na serwer i napisz !ping. Bot powinien odpisać!

ROZDZIAŁ 6: COŚ NIE DZIAŁA?

  • Błąd "Disallowed Intents"? Zapomniałeś włączyć przełączników w Rozdziale 2.
  • Błąd "Token Invalid"? Źle wkleiłeś token w Rozdziale 4.
  • Bot nie odpisuje? Upewnij się, że zapisałeś plik (Ctrl + S) i zrestartowałeś bota (Ctrl + C w terminalu i wpisz node index.js ponownie).

ROZDZIAŁ 7: KOMENDY SLASH (/)

Nowoczesne boty używają ukośnika. Nauczmy bota komendy /del, która kasuje wiadomości. Komendy wymagają specjalnego radaru, zwanego interactionCreate.

KROK 1: Na samej górze pliku zamień pierwszą linijkę na tę, która dodaje nowe narzędzie:

const { Client, GatewayIntentBits, ApplicationCommandOptionType } = require('discord.js');

KROK 2: Zarejestruj komendę dla Discorda. Wewnątrz bloku client.on('ready', () => { ... }) pod napisem console.log dopisz to:

   client.application.commands.create({
        name: 'del',
        description: 'Usuwa wiadomosci',
        options: [{
            name: 'ilosc',
            description: 'Ile usunac?',
            type: ApplicationCommandOptionType.Integer,
            required: true
        }]
    });

KROK 3: Teraz powiedz botowi, co ma zrobić. Nad linijką client.login(TOKEN) (czyli na samym dole, przed końcem pliku) wklej radar na komendy:

client.on('interactionCreate', async (interaction) => {
    if (!interaction.isChatInputCommand()) return;
    if (interaction.commandName === 'del') {
        const liczba = interaction.options.getInteger('ilosc');
        await interaction.channel.bulkDelete(liczba);
        await interaction.reply({ content: `Usunieto ${liczba} wiadomosci!`, ephemeral: true });
    }
});

CZĘŚĆ 2: BAZY DANYCH I ROZBUDOWANIE BOTA

Zwykłe zmienne w kodzie znikają, gdy zrestartujesz bota. Pora nauczyć naszego wirtualnego pracownika zapisywania danych do plików, tworząc trwałą ekonomię i sklep!

ROZDZIAŁ 8: TRYB DEWELOPERA (KLUCZ DO SKLEPU)

Aby bot wiedział, komu nadać rolę lub który produkt kupić w sklepie, będziesz potrzebować numerów ID (długie ciągi cyfr). Discord domyślnie je ukrywa, więc musisz je włączyć!

  • Otwórz Discorda i wejdź w Ustawienia Użytkownika (zębatka na dole).
  • Zjedź po lewej stronie do zakładki Zaawansowane.
  • Włącz suwak przy opcji Tryb dewelopera.
  • Co to daje? Od teraz, gdy klikniesz na dowolną rolę, kanał lub użytkownika prawym przyciskiem myszy, pojawi się opcja: "Kopiuj ID".

ROZDZIAŁ 9: TRWAŁA PAMIĘĆ (PLIKI .JSON)

Stworzymy plik do trzymania monet (db.json) oraz asortyment naszego sklepu z rolami (sklep.json).

KROK 1: Plik z monetami
W swoim folderze bota stwórz nowy plik o nazwie db.json. Wpisz w nim tylko dwie klamry: {} i zapisz plik. Tu bot będzie trzymał monety.

KROK 2: Plik ze sklepem
Stwórz drugi plik o nazwie sklep.json. Wklej tam skopiowane (dzięki trybowi dewelopera) numery ID ról ze swojego serwera i nadaj im ceny:

{
  "WKLEJ_TUTAJ_ID_ROLI_1": 50,
  "WKLEJ_TUTAJ_ID_ROLI_2": 150,
  "WKLEJ_TUTAJ_ID_ROLI_3": 500
}

ROZDZIAŁ 10: HIERARCHIA RÓL (BARDZO WAŻNE)

Zanim zaczniemy kodować sklep, musisz wiedzieć jedną kluczową rzecz: Bot nigdy nie nada roli graczowi, jeśli ta rola znajduje się wyżej w ustawieniach serwera niż rola samego bota. Wejdź w Ustawienia serwera -> Role na swoim Discordzie. Znajdź rolę swojego bota i przesuń ją na samą górę listy (nad role, które zamierzasz sprzedawać). Inaczej bot będzie sypał błędami!

ROZDZIAŁ 11: TWORZYMY SKLEP (KROK PO KROKU)

Zanim zbudujesz pełen sklep, musisz zrozumieć jak działa zaktualizowany radar bota. Od teraz nasz radar interactionCreate musi obsługiwać dwie osobne rzeczy: komendy wpisywane z klawiatury (Slash) oraz kliknięcia w Menu rozwijane.

Na samej górze pliku index.js nie zapomnij dodać narzędzi do czytania plików i tworzenia menu! W pierwszej linijce dodaj StringSelectMenuBuilder do nawiasów, a pod spodem dopisz nową linijkę: const fs = require('fs');

KROK 1: Wczytywanie bazy danych
Na samym początku radaru interactionCreate musimy dodać bezpieczny system wczytywania portfela gracza:

    // --- BEZPIECZNE WCZYTANIE BAZY DANYCH ---
    let db;
    try { 
        db = JSON.parse(fs.readFileSync('./db.json', 'utf-8')); 
    } catch (e) { 
        db = {}; 
    } 
    
    let uid = interaction.user.id;
    if (!db[uid]) db[uid] = 0; // Jeśli gracz jest nowy, ustaw mu 0 na start

KROK 2: Sekcja Komend Slash (Wyświetlenie sklepu)
Następnie otwieramy blok dla komend slash za pomocą if (interaction.isChatInputCommand()) {. Wewnątrz niego dodajemy komendę, która czyta z pliku sklep.json, odnajduje prawdziwe nazwy ról na serwerze i tworzy z nich menu wyboru dla gracza:

        // KOMENDA: /sklep
        if (interaction.commandName === 'sklep') {
            const towary = JSON.parse(fs.readFileSync('./sklep.json', 'utf-8'));
            const menu = new StringSelectMenuBuilder().setCustomId('sklep_menu').setPlaceholder('Kup role...');
            
            Object.keys(towary).forEach(id => { 
                const rolaNaSerwerze = interaction.guild.roles.cache.get(id);
                const nazwaRoli = rolaNaSerwerze ? rolaNaSerwerze.name : 'BŁĄD ID ROLI';
                menu.addOptions({ label: `Rola: ${nazwaRoli}`, value: id, description: `Cena: ${towary[id]} monet` }); 
            });
            await interaction.reply({ content: 'Wybierz role:', components: [new ActionRowBuilder().addComponents(menu)], ephemeral: true });
        }

KROK 3: Sekcja Menu (Zakup roli)
Gdy gracz wybierze rolę z rozwijanej listy, to już nie jest komenda Slash! Wychodzimy poza blok isChatInputCommand i dodajemy nowy warunek: isStringSelectMenu. To tutaj bot sprawdza stan konta, nadaje rolę i zapisuje zmiany do bazy danych:

    // --- SEKCJA MENU WYBORU (Obsługa kliknięcia w opcję w /sklep) ---
    if (interaction.isStringSelectMenu() && interaction.customId === 'sklep_menu') {
        const rolaId = interaction.values[0];
        const towary = JSON.parse(fs.readFileSync('./sklep.json', 'utf-8'));
        
        if (db[uid] < towary[rolaId]) return interaction.reply({ content: 'Masz za malo kasy!', ephemeral: true });
        
        const rolaDoNadania = interaction.guild.roles.cache.get(rolaId);
        if (!rolaDoNadania) return interaction.reply({ content: 'Blad: Taka rola nie istnieje na serwerze!', ephemeral: true });
        
        try {
            await interaction.member.roles.add(rolaDoNadania);
            db[uid] -= towary[rolaId];
            fs.writeFileSync('./db.json', JSON.stringify(db, null, 2)); 
            await interaction.reply({ content: `Udalo sie! Kupiono role **${rolaDoNadania.name}**.`, ephemeral: true });
        } catch (error) {
            await interaction.reply({ content: `**BŁĄD!** Bot nie może nadać tej roli. Przesuń rolę bota ponad role sklepowe w ustawieniach Serwera!`, ephemeral: true });
        }
    }

ROZDZIAŁ 12: PEŁNY KOD (PUNKT KONTROLNY)

Wszystko w jednym miejscu! Jeśli w trakcie nauki twój kod przestał działać, pogubiłeś się w układaniu klamerek, wyskakują błędy w terminalu, lub po prostu chcesz mieć gotowy, w pełni funkcjonujący kod łączący wszystkie nasze lekcje w jedną spójną całość – znajdziesz go poniżej. Skopiuj ten kod, zastąp nim całą zawartość swojego pliku index.js i baw się dobrze! 
// 1. IMPORTOWANIE NARZĘDZI I KONFIGURACJA
const { Client, GatewayIntentBits, ApplicationCommandOptionType, ActionRowBuilder, ButtonBuilder, ButtonStyle, StringSelectMenuBuilder } = require('discord.js');
const fs = require('fs');

const TOKEN = 'TUTAJ_WKLEJ_SWOJ_TAJNY_TOKEN';

// 2. TWORZENIE BOTA
const client = new Client({
    intents: [
        GatewayIntentBits.Guilds,
        GatewayIntentBits.GuildMessages,
        GatewayIntentBits.MessageContent
    ]
});

// 3. KIEDY BOT SIE URUCHOMI... (REJESTRACJA KOMEND)
client.on('ready', () => {
    console.log(`Zrobione! Bot zalogowal sie jako: ${client.user.tag}`);
    client.application.commands.create({ name: 'eko', description: 'Graj w karty' }); 
    client.application.commands.create({ name: 'portfel', description: 'Stan konta' });
    client.application.commands.create({ name: 'sklep', description: 'Kup role' });
    client.application.commands.create({
        name: 'del',
        description: 'Usuwa wiadomosci',
        options: [{
            name: 'ilosc',
            description: 'Ile usunac?',
            type: ApplicationCommandOptionType.Integer,
            required: true
        }]
    });
});

// 4. KIEDY KTOS NAPISZE ZWYKLA WIADOMOSC...
client.on('messageCreate', (message) => {
    if (message.author.bot) return;

    if (message.content === '!ping') {
        message.reply('Pong! Moj pierwszy kod dziala!');
    }
});

// 5. RADAR NA INTERAKCJE (SLASH, PRZYCISKI I MENU)
client.on('interactionCreate', async (interaction) => {
    
    // --- BEZPIECZNE WCZYTANIE BAZY DANYCH ---
    let db;
    try { 
        db = JSON.parse(fs.readFileSync('./db.json', 'utf-8')); 
    } catch (e) { 
        db = {}; 
    } 
    
    let uid = interaction.user.id;
    if (!db[uid]) db[uid] = 0; // Jeśli gracz jest nowy, ustaw mu 0 na start

    // --- SEKCJA A: KOMENDY SLASH (Wpisywane z klawiatury) ---
    if (interaction.isChatInputCommand()) {
        
        // KOMENDA: /sklep
        if (interaction.commandName === 'sklep') {
            const towary = JSON.parse(fs.readFileSync('./sklep.json', 'utf-8'));
            const menu = new StringSelectMenuBuilder().setCustomId('sklep_menu').setPlaceholder('Kup role...');
            
            Object.keys(towary).forEach(id => { 
                const rolaNaSerwerze = interaction.guild.roles.cache.get(id);
                const nazwaRoli = rolaNaSerwerze ? rolaNaSerwerze.name : 'BŁĄD ID ROLI';
                menu.addOptions({ label: `Rola: ${nazwaRoli}`, value: id, description: `Cena: ${towary[id]} monet` }); 
            });
            await interaction.reply({ content: 'Wybierz role:', components: [new ActionRowBuilder().addComponents(menu)], ephemeral: true });
        }

        // KOMENDA: /del
        if (interaction.commandName === 'del') {
            const liczba = interaction.options.getInteger('ilosc');
            await interaction.channel.bulkDelete(liczba);
            await interaction.reply({ content: `Usunieto ${liczba} wiadomosci!`, ephemeral: true });
        }
        
        // KOMENDA: /portfel
        if (interaction.commandName === 'portfel') {
            let kasa = db[uid]; 
            await interaction.reply(`Masz ${kasa} monet.`);
        }

        // KOMENDA: /eko (KASYNO Z PRZYCISKAMI)
        if (interaction.commandName === 'eko') {
            let karta1 = Math.floor(Math.random() * 10) + 1;
            const rzad = new ActionRowBuilder().addComponents(
                new ButtonBuilder().setCustomId('ryzykuj').setLabel('Ryzykuj!').setStyle(ButtonStyle.Danger),
                new ButtonBuilder().setCustomId('odbierz').setLabel(`Zabierz ${karta1}`).setStyle(ButtonStyle.Success)
            );

            const msg = await interaction.reply({ content: `Masz karte: ${karta1}. Ryzykujesz?`, components: [rzad] });
            const kol = msg.createMessageComponentCollector({ time: 30000 });

            kol.on('collect', async (i) => {
                if (i.user.id !== interaction.user.id) return i.reply({ content: 'To nie Twoja gra!', ephemeral: true });
                
                if (i.customId === 'odbierz') { 
                    db[uid] += karta1; 
                    await i.update({ content: `Zabrales ${karta1} monet!`, components: [] });
                } else { 
                    let karta2 = Math.floor(Math.random() * 10) + 1;
                    if (karta2 > karta1) {
                        let win = (karta2 === 10) ? 100 : karta2;
                        db[uid] += win; 
                        await i.update({ content: `WYGRANA! Masz ${karta2}. Zgarniasz ${win} monet!`, components: [] });
                    } else {
                        await i.update({ content: `PRZEGRANA! Masz ${karta2}. Tracisz wszystko.`, components: [] });
                    }
                }
                
                // Po grze nadpisujemy plik ze zmienionymi monetami:
                fs.writeFileSync('./db.json', JSON.stringify(db, null, 2));
                kol.stop();
            });
        }
    }

    // --- SEKCJA B: MENU WYBORU (Obsługa kliknięcia w opcję w /sklep) ---
    if (interaction.isStringSelectMenu() && interaction.customId === 'sklep_menu') {
        const rolaId = interaction.values[0];
        const towary = JSON.parse(fs.readFileSync('./sklep.json', 'utf-8'));
        
        if (db[uid] < towary[rolaId]) return interaction.reply({ content: 'Masz za malo kasy!', ephemeral: true });
        
        const rolaDoNadania = interaction.guild.roles.cache.get(rolaId);
        if (!rolaDoNadania) return interaction.reply({ content: 'Blad: Taka rola nie istnieje na serwerze!', ephemeral: true });
        
        // Zabezpieczenie przed błędem hierarchii ról z Rozdziału 10
        try {
            await interaction.member.roles.add(rolaDoNadania);
            db[uid] -= towary[rolaId];
            fs.writeFileSync('./db.json', JSON.stringify(db, null, 2)); 
            await interaction.reply({ content: `Udalo sie! Kupiono role **${rolaDoNadania.name}**.`, ephemeral: true });
        } catch (error) {
            await interaction.reply({ content: `**BŁĄD!** Bot nie może nadać tej roli. Przesuń rolę bota ponad role sklepowe w ustawieniach Serwera!`, ephemeral: true });
        }
    }

});

// 6. LOGOWANIE BOTA
client.login(TOKEN);

ZADANIE: KOMENDA /PRACA

Zanim przejdziesz do słownika, spróbuj samodzielnie stworzyć nową komendę o nazwie /praca. Wymagania:

  1. Po jej użyciu bot ma wysłać wiadomość z zielonym przyciskiem o napisie "Pracuj".
  2. Kiedy użytkownik w niego kliknie, bot ma zaktualizować bazę danych dodając mu 5 monet (db[uid] += 5) i zapisać plik z bazą (fs.writeFileSync...).
  3. Bot ma odpisać po udanej pracy: "Zarobiłeś 5 coinów!".

PEŁNY SŁOWNIK POJĘĆ

  • const: Tworzy pudełko z informacją, której bot nie może zmienić w trakcie pracy.
  • require: Komenda, która pobiera odpowiednią instrukcję/paczkę narzędzi (np. discord.js lub moduł plików fs).
  • Client / Intents: Główny mózg bota oraz zestaw pozwoleń, na co bot może patrzeć.
  • client.on('ready'): Radar, który alarmuje, gdy tylko bot włączy się i jest gotowy do pracy.
  • client.on('messageCreate'): Radar, który budzi się, kiedy ktokolwiek napisze zwykłą wiadomość na kanale.
  • client.on('interactionCreate'): Nowoczesny radar, obsługujący komendy ukośnikowe (/), przyciski oraz menu wyboru.
  • interaction.reply: Najprostszy sposób, by bot odpowiedział konkretnej osobie po wpisaniu komendy.
  • ephemeral: true: Sekretna wiadomość. Pojawia się na ekranie tylko osobie, która nacisnęła przycisk, dla reszty serwera jest niewidoczna.
  • fs.readFileSync / fs.writeFileSync: Magiczne polecenia, które pozwalają botowi fizycznie czytać i zapisywać dane do plików (np. .json) na twoim komputerze.
  • JSON.parse: Tłumacz tekstów. Zamienia linijki tekstu z pliku .json na kod, na którym bot może pracować.
  • JSON.stringify: Tłumacz powrotny. Bierze dane z pamięci bota i zamienia je w tekst gotowy do wklejenia w plik.
  • try { ... } catch (error) { ... }: Kamizelka ratunkowa dla bota. Jeśli kod w sekcji "try" wyrzuci błąd (np. brak pliku lub brak uprawnień), bot nie wyłączy się z hukiem, tylko przejdzie do sekcji "catch" i wykona bezpieczną alternatywę.
  • interaction.options.getInteger: Sprawdza, jaką liczbę ktoś wpisał korzystając z komendy (np. z komendą /del).
  • bulkDelete: Służy do masowego kasowania wielu wiadomości z czatu jednocześnie.
  • ActionRowBuilder: "Szufladka" na komponenty. Żaden przycisk ani menu nie może istnieć samodzielnie, muszą być do niej włożone.
  • ButtonBuilder: Służy do tworzenia i personalizowania przycisków.
  • ButtonStyle: Styluje kolor przycisku (Success = Zielony, Danger = Czerwony).
  • StringSelectMenuBuilder: Tworzy rozwijaną listę (tę, z której wybieramy role w sklepie).
  • createMessageComponentCollector: Stoper z okiem. Bot czeka przez określony w kodzie czas na kliknięcie stworzonego elementu.
  • interaction.member.roles.add: Magiczna różdżka nadająca wybraną rolę użytkownikowi. Wymaga odpowiedniej hierarchii ról na serwerze!
  • interaction.guild.roles.cache.get(): Funkcja, która przeszukuje serwer po numerze ID i wyciąga z niego wszystkie informacje o roli (np. jej nazwę).
  • Object.keys(): Polecenie, które wyciąga wszystkie nazwy własne (np. numery ID ról) z pliku sklepu i wypisuje je w liście menu.
  • Math.random / Math.floor: Maszyna losująca liczby dziesiętne połączona z funkcją, która zaokrągla ułamek do pełnej liczby. W taki sposób uzyskaliśmy system szans.
